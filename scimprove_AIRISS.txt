# 🧠 AIRISS 평가의견 분석 기능 개발 작업지시서 (Claude Code용)

## 📌 1. 프로젝트 개요

- **프로젝트명**: AIRISS 텍스트기반 평가의견 분석 확장
- **목표**: 엑셀로 수집된 직원 평가의견 데이터를 정제, 요약, 점수화하여 AIRISS 성과/역량 스코어링에 통합
- **대상 데이터**: 최근 10년치 평가의견 (연도별 다건 존재 가능, 결측 존재)
- **출력물**: 정제된 평가 요약, 강점/약점 키워드, 텍스트 점수, 조직 역량 매핑 결과

---

## 📋 2. PRD (Product Requirements Document)

### 2.1 주요 기능 요약

| 기능 | 설명 |
|------|------|
| 평가의견 업로드 | .xlsx/.csv → UID 및 의견 컬럼 자동 감지 |
| 연도 병합 & 누락 처리 | UID 기준 연도별 병합. 결측 연도는 “의견 없음” 표시 |
| 평가의견 정제 | 특수문자 제거, 정규화, 공백 처리 등 |
| LLM 요약 | 평가의견을 1~2문장으로 요약 |
| 키워드 추출 | 강점/약점 키워드 리스트 추출 |
| 텍스트 점수화 | 긍정도/구체성/난이도 등을 기반으로 점수화 |
| 역량 프레임워크 매핑 | AIRISS 8대 조직역량 기준으로 평가내용 매핑 |
| hybrid_score 계산 | 정량점수 + 텍스트점수를 가중 평균하여 통합 |
| 분석결과 저장 | EmployeeResult 테이블 확장 (text, hybrid 관련 필드 포함) |
| 프론트 시각화 | 평가요약/강약점/스코어 등을 UI에 노출 |
