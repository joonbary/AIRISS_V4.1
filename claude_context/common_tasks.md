# AIRISS 자주 하는 작업들 (Common Tasks)

## 🎯 가장 빈번한 작업 유형

### 1. 📊 새로운 분석 기능 추가
```
빈도: ★★★★★ (매주)
복잡도: 중간-높음
소요 시간: 2-5일
주요 패턴: 모듈 생성 → 통합 → 테스트 → UI 추가
```

### 2. 🐛 버그 수정 및 성능 최적화
```
빈도: ★★★★☆ (주 2-3회)
복잡도: 낮음-중간
소요 시간: 1-2일
주요 패턴: 재현 → 분석 → 수정 → 검증
```

### 3. 🎨 UI/UX 개선
```
빈도: ★★★☆☆ (주 1회)
복잡도: 낮음-중간
소요 시간: 0.5-2일
주요 패턴: 디자인 → 구현 → 반응형 → 테스트
```

### 4. 🔧 설정 및 배포 관련
```
빈도: ★★☆☆☆ (격주)
복잡도: 중간
소요 시간: 0.5-1일
주요 패턴: 설정 변경 → 테스트 → 배포 → 검증
```

## 📋 작업별 상세 가이드

### 🧩 Task 1: 새로운 분석 차원 추가

#### 🎯 목표 예시
"9번째 차원으로 '디지털 역량(Digital Competency)' 추가"

#### 📝 작업 단계
```markdown
### Step 1: 텍스트 분석 프레임워크 확장
파일: app/services/text_analyzer.py
수정 내용:
- framework 딕셔너리에 '디지털역량' 키워드 추가
- 관련 키워드 리스트 정의
- 가중치 설정

### Step 2: 정량 분석 로직 추가
파일: app/services/quantitative_analyzer.py
수정 내용:
- 디지털 관련 지표 분석 함수 추가
- 점수 계산 로직 구현

### Step 3: 하이브리드 분석 통합
파일: app/services/hybrid_analyzer.py
수정 내용:
- comprehensive_analysis() 함수에 새 차원 통합
- 결과 구조체에 디지털역량 점수 추가

### Step 4: 프론트엔드 시각화 추가
파일: airiss-v4-frontend/src/components/Charts/RadarChart.jsx
수정 내용:
- 레이더 차트에 9번째 축 추가
- 색상 및 라벨 설정

### Step 5: 데이터베이스 스키마 확장
파일: app/models/analysis_result.py
수정 내용:
- dimension_scores JSON에 디지털역량 필드 추가
- 마이그레이션 스크립트 작성 (필요시)
```

#### ✅ 완료 체크리스트
```
- [ ] 기존 8대 차원 분석 정상 동작
- [ ] 새 차원 점수 계산 정확성 확인
- [ ] 레이더 차트 9개 축 정상 표시
- [ ] 전체 분석 플로우 정상 동작
- [ ] 성능 저하 없음 확인
```

### 🔍 Task 2: 편향 탐지 알고리즘 개선

#### 🎯 목표 예시
"성별 편향 탐지 정확도를 80% → 90%로 향상"

#### 📝 작업 단계
```markdown
### Step 1: 현재 편향 탐지 로직 분석
파일: app/services/bias_detection/bias_detector.py
분석 내용:
- 현재 4/5 규칙 적용 방식 검토
- 통계적 유의성 검정 추가 필요성 검토
- 표본 크기 고려사항 검토

### Step 2: 개선된 알고리즘 구현
새 파일: app/services/bias_detection/enhanced_bias_detector.py
구현 내용:
- 다중 통계 테스트 적용
- 교차 검증 로직
- 신뢰구간 계산
- 효과 크기 측정

### Step 3: 성능 벤치마킹
테스트 파일: tests/test_bias_detection_accuracy.py
테스트 내용:
- 합성 데이터로 정확도 측정
- 실제 데이터로 검증
- 기존 vs 개선된 알고리즘 비교

### Step 4: 점진적 배포
배포 전략:
- A/B 테스트로 10% 트래픽에 적용
- 정확도 모니터링
- 점진적 확대 (50% → 100%)
```

#### ✅ 완료 체크리스트
```
- [ ] 기존 편향 탐지 기능 정상 동작
- [ ] 새 알고리즘 정확도 90% 이상
- [ ] 기존 API 호환성 유지
- [ ] 성능 저하 없음 (응답시간 동일)
- [ ] 사용자 인터페이스 정상 동작
```

### 📈 Task 3: 성능 최적화

#### 🎯 목표 예시
"대용량 파일(2000+ 레코드) 분석시 메모리 사용량 50% 절약"

#### 📝 작업 단계
```markdown
### Step 1: 성능 병목 지점 분석
도구: Python profiler, memory_profiler
분석 파일:
- app/services/hybrid_analyzer.py
- app/services/text_analyzer.py
- app/services/excel_service.py

분석 항목:
- 메모리 사용량 피크 지점
- CPU 집약적 함수들
- I/O 대기 시간

### Step 2: 배치 처리 로직 구현
새 파일: app/services/batch_processor.py
구현 내용:
- 청크 단위 데이터 처리
- 메모리 효율적 알고리즘
- 진행률 실시간 업데이트

### Step 3: 캐싱 시스템 도입
새 파일: app/services/cache_manager.py
구현 내용:
- LRU 캐시로 분석 결과 저장
- 중복 분석 방지
- 메모리 기반 임시 캐싱

### Step 4: 가비지 컬렉션 최적화
수정 파일: app/services/hybrid_analyzer.py
수정 내용:
- 명시적 del 명령어 추가
- 순환 참조 방지
- 메모리 누수 제거
```

#### ✅ 완료 체크리스트
```
- [ ] 메모리 사용량 50% 감소
- [ ] 처리 속도 동일 수준 유지
- [ ] 대용량 파일 처리 안정성
- [ ] 기존 기능 정상 동작
- [ ] 메모리 누수 없음
```

- 팀별 평균 점수 계산
